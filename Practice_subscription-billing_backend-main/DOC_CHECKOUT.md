# Сценарий: Оформление подписки клиентом

## 1. АЛГОРИТМ РАБОТЫ

1. Получить `tariff_id` от клиентского приложения через POST-запрос на эндпоинт `/subscriptions`. ID текущего пользователя извлекается из JWT-токена.

2. Проверить ограничение на число одновременно активных подписок (FR-7):
   - ЕСЛИ у пользователя уже есть подписка со статусом "active" (активна) или "trial" (пробная), ТО вернуть ошибку с кодом 400 "У вас уже есть активная подписка".
   - ИНАЧЕ продолжить выполнение.

3. Запросить из БД данные тарифа: `price`, `trial_days`, `period_months`.

4. Сформировать даты и статус подписки (FR-6, FR-9):
   - ЕСЛИ `trial_days` > 0, ТО установить статус подписки = "trial" (пробная), дату следующего списания = текущая дата + `trial_days` дней.
   - ИНАЧЕ установить статус подписки = "active" (активна), выполнить списание на сумму `price`, установить дату следующего списания = текущая дата + `period_months` месяцев.

5. Сформировать итоговую запись подписки в базе данных по правилам части «Преобразования».

6. Вернуть клиенту объект созданной подписки с датой следующего списания.

## 2. ТИПЫ ДАННЫХ

| № | НАЗВАНИЕ ПОЛЯ | ТИП |
| :--- | :--- | :--- |
| 01 | user_id (из токена) | целое |
| 02 | tariff_id | целое |
| 03 | price | дробное |
| 04 | trial_days | целое |
| 05 | period_months | целое |
| 06 | status | перечисление (active, trial, cancelled, paused, expired) |
| 07 | next_billing_date | дата-время |
| 08 | start_date | дата-время |
| 09 | trial_end_date | дата-время (опционально) |

## 3. ПРЕОБРАЗОВАНИЯ

| ИСХОДНЫЕ ДАННЫЕ (ПОЛЕ) | ТИП | ПРАВИЛО ПРЕОБРАЗОВАНИЯ | ИТОГОВЫЕ ДАННЫЕ (ПОЛЕ) | ТИП |
| :--- | :--- | :--- | :--- | :--- |
| tariff_id | целое | Найти тариф в БД и получить `trial_days`, `period_months`, `price` | trial_days, period_months, price | целое, дробное |
| trial_days | целое | ЕСЛИ `trial_days` > 0 ТО "trial" ИНАЧЕ "active" | status | перечисление |
| trial_days | целое | Вычислить: `start_date` + `trial_days` дней | trial_end_date | дата-время |
| period_months | целое | ЕСЛИ статус "active" ТО вычислить: `start_date` + `period_months` месяцев | next_billing_date | дата-время |
| системное время | дата-время | Присвоить текущее время сервера | start_date | дата-время |
