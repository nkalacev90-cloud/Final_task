# Сценарий: Повторные попытки списания при ошибке оплаты (Dunning)

## 1. АЛГОРИТМ РАБОТЫ

1. Серверный планировщик (Cron или APScheduler) автоматически запускает скрипт обработки неудавшихся платежей каждые сутки.

2. Скрипт выбирает из БД все подписки, у которых:
   - `status` = "payment_error" (ошибка оплаты)
   - И `next_retry_date` <= Текущая дата-время сервера.

3. ДЛЯ КАЖДОЙ найденной подписки выполнить:
   - Проверить текущее количество совершенных попыток (`retry_count`).

   - ЕСЛИ `retry_count` >= 3 (достигнут лимит попыток), ТО:
     1. Изменить статус подписки на "paused" (приостановлена).
     2. Записать в лог окончательный отказ в проведении платежа.

   - ИНАЧЕ (попытки еще остались):
     1. Вызвать функцию имитации списания денег со счета на сумму `price` (из тарифа).
     2. Проверить результат списания:
        - ЕСЛИ списание успешно, ТО:
          - Изменить статус подписки на "active" (активна).
          - Сбросить счетчик попыток: `retry_count` = 0.
          - Обновить дату следующего планового списания: `next_billing_date` = текущая дата + `period_months` месяцев.
          - Создать запись в таблице `payments` со статусом "paid".
        - ИНАЧЕ (списание опять не прошло):
          - Увеличить счетчик попыток: `retry_count` = `retry_count` + 1.
          - Назначить дату новой попытки: `next_retry_date` = текущая дата + 1 день.
          - Создать запись в таблице `payments` со статусом "failed".

4. Завершить работу скрипта и записать лог выполнения.

## 2. ТИПЫ ДАННЫХ

| № | НАЗВАНИЕ ПОЛЯ | ТИП |
| :--- | :--- | :--- |
| 01 | subscription_id | целое |
| 02 | status | перечисление (active, trial, cancelled, paused, expired, payment_error) |
| 03 | retry_count | целое |
| 04 | next_retry_date | дата-время |
| 05 | price | дробное |
| 06 | period_months | целое |

## 3. ПРЕОБРАЗОВАНИЯ

| ИСХОДНЫЕ ДАННЫЕ (ПОЛЕ) | ТИП | ПРАВИЛО ПРЕОБРАЗОВАНИЯ | ИТОГОВЫЕ ДАННЫЕ (ПОЛЕ) | ТИП |
| :--- | :--- | :--- | :--- | :--- |
| retry_count | целое | При неудачной оплате вычислить: retry_count + 1 | retry_count | целое |
| retry_count | целое | ЕСЛИ retry_count >= 3 ТО "paused" ИНАЧЕ оставить "payment_error" | status | перечисление |
| системное время | дата-время | При неудачной оплате вычислить: текущее время + 1 день | next_retry_date | дата-время |
| retry_count | целое | При успешной оплате присвоить 0 | retry_count | целое |
| price | дробное | Записать сумму успешного списания в платеж | amount (в payments) | дробное |
