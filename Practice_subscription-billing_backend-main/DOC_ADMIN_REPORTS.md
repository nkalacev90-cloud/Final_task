# Сценарий: Генерация финансовых отчетов для администратора

## 1. АЛГОРИТМ РАБОТЫ

1. Получить параметры `start_date` и `end_date` от клиентского приложения администратора через GET-запрос на эндпоинт `/admin/revenue`.

2. Проверить корректность введенных дат:
   - ЕСЛИ `start_date` больше, чем `end_date`, ТО вернуть ошибку с кодом 400 "Некорректный временной интервал".
   - ИНАЧЕ продолжить выполнение.

3. Сделать запрос к таблице `payments` в БД через SQLAlchemy:
   - Найти все записи со статусом `"paid"`, у которых дата платежа (`created_at`) находится в диапазоне от `start_date` до `end_date`.
   - Просуммировать значения полей `amount` для этих записей и записать результат в переменную `total_revenue`.

4. Сделать запрос к таблице `subscriptions` в БД:
   - Посчитать общее количество записей, а также количество записей, сгруппированных по их статусам (`status`).

5. Сформировать итоговый JSON-объект отчета, содержащий:
   - Общую выручку (`total_revenue`)
   - Количество успешных платежей (`successful_payments_count`)
   - Количество неуспешных платежей (`failed_payments_count`)
   - Общее количество активных подписок (`active_subscriptions_count`)
   - Количество новых подписок за период (`new_subscriptions_count`)

6. Вернуть сформированный отчет администратору с кодом 200.

---

## 2. ТИПЫ ДАННЫХ

| № | НАЗВАНИЕ ПОЛЯ | ТИП |
| :--- | :--- | :--- |
| 01 | start_date | дата-время |
| 02 | end_date | дата-время |
| 03 | total_revenue | дробное |
| 04 | status | перечисление (paid, failed, pending) |
| 05 | successful_payments_count | целое |
| 06 | failed_payments_count | целое |
| 07 | active_subscriptions_count | целое |
| 08 | new_subscriptions_count | целое |
